#  MetaAgent — 自进化元智能体架构

## 第十九届全国大学生软件创新大赛 · 项目构思总结

> **赛题方向：** 赛题2 · 随身AI互动导学
>
> **核心一句话：** "我们做的不是又一个AI工具，而是一个替代'你作为AI调度者'的元Agent。它不和ChatGPT/Claude/Codex竞争——它调度它们。"

---

## 一、痛点洞察

大学生现在的AI工作流：
- 打开Claude Code写代码 → 打开ChatGPT问实验报告 → 打开Gemini查资料 → 打开NotebookLM整理笔记……
- **人类自己成了"AI调度中枢"**：整理需求 → 问AI → 整理结果 → 再问另一个AI → 重复
- 每次切换AI都要**从头补充上下文**，极其低效
- 现有Agent做错了会**一错再错**，浪费时间让人骂娘

**我们的解决方案：** 在所有AI工具之上封装一层"元Agent"，替代"人作为调度者"的角色，让AI帮你调度AI。

---

## 二、产品定位

### 与现有产品的错位竞争

| 维度 | NotebookLM | ChatGPT/Claude | **我们的MetaAgent** |
|------|-----------|---------------|-------------------|
| 本质 | 被动文档分析器 | 单轮/多轮对话工具 | **主动任务执行者+调度者** |
| 记忆 | 无跨会话记忆 | 有限上下文 | **持久记忆库，越用越懂你** |
| 学习 | 不学习用户习惯 | 不学习 | **从示范中学习，封装Skill** |
| 操作 | 只能分析文档 | 只能对话 | **可操控软件、执行任务** |
| 交互 | 单向（你问它答） | 单向 | **双向（它也能主动找你）** |

**核心叙事：NotebookLM帮你"理解"知识，我们的Agent帮你"完成"任务。我们的Agent甚至可以通过UI Agent去操控NotebookLM——因为我们在更高的抽象层级上运作。**

## 三、架构设计

### 总体思路
**一套通用内核架构（开源） + 两个赛题专精模块（比赛）**

架构本身是通用的、可扩展的，任何人都可以给GitHub贡献新模块。比赛聚焦做深两个核心模块。不怕新工具/新SOTA出现，因为我们的Agent就是来调用它们的。

### 🧠 内核架构（三件套）

#### 1. 自学习引擎（Self-Learning Engine）
- **Learning from Demonstration（从示范中学习）**：用户操作一遍，Agent观察并学会
- **Skill Library（技能库）**：学到的操作封装成可复用的Skill函数
- **Learning Policy from Feedback（从反馈中学习）**：做错了→谦虚请教用户→用户示范正确做法→封装新Skill→不再犯同样的错
- **元策略学习**：不仅学会"怎么做"，还学会"用什么工具做、什么组合最优"（学习用户的工具协同习惯）

#### 2. 持久记忆库（Persistent Memory）
- **用户画像**：了解用户的专业、偏好、能力水平
- **知识图谱**：用户的专业知识、课程内容、项目上下文
- **定时摘要**：周期性整合对话信息，维护永久记忆
- **效果**：第一句话就能进入状态，无需每次重复补充上下文

#### 3. 模块接口标准（Module Interface Standard）
- 统一的Skill注册/调用/管理接口
- 支持MCP（Model Context Protocol）扩展
- 开放给社区贡献新模块

### 📱 专精模块1 — UI Agent
- 屏幕理解 + 模拟操作（参考Mobile-Agent v3 / DroidRun）
- 可操控手机/电脑上的各种应用
- 包括操控其他AI工具（浏览器里的ChatGPT/Gemini等）

### 📚 专精模块2 — AI智能学习助手
- **知识整理**：文件夹 → 磁贴化输出（md / pptx / word / pdf）
- **实验辅助**：帮你跑代码、整理数据、写实验报告
- **智能答疑**：简单问题直接答，复杂问题调用更强模型

## 四、核心交互设计

### 1. Chat / Agent 双模式智能切换
- **Chat模式**：日常聊天、快速问答、知识查询
- **Agent模式**：复杂任务执行、多步骤操作、跨应用协同
- **智能路由**：Agent自己判断任务复杂度，主动推荐切换模式
- 例：用户说"帮我打开明日方舟签到" → Agent推荐切换Agent模式 → 展示执行Plan → 用户确认后执行

### 2. Plan预览 + 批红机制（"皇帝批红"模式）
- Agent接到复杂任务 → **先出Plan给用户审批**
- Plan包括：采用什么方案、需要哪些工具、预计耗时、可能的替代方案
- 例："我搜到了网易云API，用这个如何？只写前端/xx，需要用你电脑的这些工具，预计耗时30分钟"
- 用户批准/修改 → Agent开始执行

### 3. 进度聊天 + 中途变向
- 复杂任务执行期间，Agent**持续汇报进度**，像朋友一样跟你聊
- 用户可以**随时中途改方向**："等等，换个loss function试试"
- 也支持**挂后台静默执行**，完成后通知
- 核心目的：消除等待焦虑，保持用户掌控感，即使做错了也不会骂娘

### 4. 回调心跳机制（Hook Callback）
- 长任务完成后 → 钩子函数通知Agent → 继续后处理
- **省Token**：长任务期间不需要保持对话流
- **双向对话**：Agent可以主动发起对话
  - "早上好！你昨天的CNN实验跑完了，accuracy 94.2%"
  - "你今天下午有软件体系结构课，要我提前整理笔记吗？"

### 5. 谦虚学习 + 自我修正
- Agent做错了 → **主动承认可能做错** → 请教用户正确做法
- 用户示范一遍 → Agent封装Skill → 永不再犯
- 与"自大到底一错再错"的现有Agent形成鲜明对比

### 6. 多Agent/多工具协同
- 单Agent学会编排多工具：先Codex规划 → Claude Code编码 → 多模型交叉验证
- 学习用户的协同策略，**自己也学会了这套工作流**
- 未来扩展：多人Agent协同（小组作业场景）

## 五、技术架构

### 跨设备通信
- **手机端**：Compose App（Agent交互界面、任务派发、进度查看）
- **电脑端**：Agent Daemon后台服务（任务执行引擎）
- **通信方式**：本地同网段局域网通信（比赛现场最稳定）

### 核心技术栈参考
- **Skill Library**：参考 SAGE（AWS, 2025.12）— RL框架+技能库自进化
- **持久记忆**：参考 Mem0 / AgeMem（2025-2026）— 图结构+层级记忆管理
- **UI Agent**：参考 Mobile-Agent v3 / DroidRun / mobile-use — 移动端GUI自动化
- **模型选择**：Gemini 3 Flash（便宜快速）为主力，复杂任务调用更强模型API

### 成本策略
- 日常Chat用轻量模型（本地部署或便宜API）
- 复杂Agent任务用强模型API
- 心跳回调机制节省长任务Token消耗
- 预期半年内本地部署模型会更强，成本持续下降

## 七、Demo剧本（杀手级现场展示）

### 🎬 "大学生的一天"（3-5分钟）

**场景1 — 早上上课（展示：持久记忆 + 主动感知 + 远程控制）**
> 打开手机App → Agent主动说"早上好，你今天有软件体系结构课，要我整理上次课的笔记吗？"
> → 用户："帮我把上课录音整理成PPT"
> → Agent展示Plan → 用户批准
> → 电脑端自动执行 → 手机推送"搞定了，PPT已存到桌面"

**场景2 — 下午实验课（展示：模式切换 + Plan审批 + 心跳回调 + 自动生成报告）**
> 手机上说"帮我跑一下今天的CNN实验"
> → Agent识别复杂任务，**建议切换Agent模式**
> → 展示Plan："我会打开你的VSCode，运行train.py，预计15分钟"
> → 用户批准 → 电脑端执行
> → 中途Agent汇报："第3个epoch了，accuracy 87%，要调参吗？"
> → 完成后回调通知 → Agent自动生成实验报告

**场景3 — 晚上复习（展示：Chat模式 + 智能答疑 + 记忆连续性）**
> 用户："帮我解释一下今天课上说的设计模式"
> → Agent直接在Chat模式回答（它记得你的课程内容）
> → 轻量解决，无需切Agent模式

## 八、开源策略 & 拓展愿景

### GitHub开源架构
- **内核三件套**开源 → 社区贡献新模块
- 模块接口标准化 → 任何人都能开发新Skill/新Agent模块
- 不怕新工具出现 → 我们的Agent就是来调用它们的

### 彩蛋模块（GitHub Star收割机）
- **Live2D模块**：AI输出表情标签 → 前端渲染Live2D表情 → Agent变成你的虚拟伙伴
- 不是比赛核心，但是开源社区差异化亮点

### 未来扩展方向
- 多人Agent协同（小组作业场景）
- 更多专精模块（健康管理、日程规划、创作辅助……）
- 本地模型部署（随着开源模型变强，成本持续下降）

## 九、注意事项 & 风险控制

### Must Have（P0）
- [ ] 核心调度引擎 + 模式切换
- [ ] 持久记忆库基础版
- [ ] 至少一个完整的Skill学习Demo
- [ ] 手机端 ↔ 电脑端通信
- [ ] 杀手级Demo场景跑通

### Nice to Have（P1）
- [ ] 完整的Skill Library管理
- [ ] UI Agent操控多个应用
- [ ] 进度聊天+中途变向
- [ ] 多工具协同编排

### 锦上添花（P2）
- [ ] Live2D表情渲染
- [ ] 多人Agent协同
- [ ] 远程异地控制

### 风险点
1. **操控浏览器AI对话**可能不稳定（反自动化/UI改版）→ 优先用API调用，浏览器操控作为展示级能力
2. **Demo现场网络**不可控 → 用本地局域网通信，避免依赖外网
3. **评委会问Token成本** → 准备成本对比表（手动做 vs Agent做的时间/金钱对比）
4. **"看一遍就学会"的边界** → 明确定义Agent能学什么类型的Skill，别让评委觉得画饼
